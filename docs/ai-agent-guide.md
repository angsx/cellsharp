# AI agent guide

This is a compact reference for humans and coding agents that generate CellSharp code. Use `using CellSharp;`; public APIs do not expose Open XML types.

## Core rules

- `Excel.Write(...)` is the convention-based path for ordinary typed exports.
- Use `Excel.Schema<T>()` when headers, order, formats, validation, formulas, tables, or worksheet settings are part of the contract. Built schemas are immutable and reusable.
- Excel cell, row, and column coordinates are one-based. `ExcelWorkbookReader.ReadAt` is the explicit exception: its public worksheet index is zero-based.
- Caller-owned streams remain open. Read streams must be readable and seekable; write streams must be writable and seekable.
- A string is always text, even when it starts with `=`, `+`, `-`, or `@`. Use a formula API only for application-authored formulas.
- CellSharp writes formulas but does not calculate them. Import uses a formula's cached value; a formula without one becomes an `InvalidValue` diagnostic.
- Header and worksheet-name matching are case-insensitive. `Excel.Open(...).ReadAt(...)` verifies that the selected sheet matches the schema name.

## Models used below

public sealed class Customer
{
    public int Id { get; set; }
    public string Name { get; set; } = "";
}

public sealed class Order
{
    public int Quantity { get; set; }

## Canonical workflows

### Write and read a list

```csharp
using CellSharp;

var customers = new[] { new Customer { Id = 1, Name = "Acme" } };
Excel.Write("customers.xlsx", customers);

var result = Excel.Read<Customer>("customers.xlsx");
foreach (var error in result.Errors)
    Console.WriteLine($"{error.CellReference}: {error.Message}");
```

### Define and reuse a schema

```csharp
var schema = Excel.Schema<Customer>()
    .SheetName("Customers")
    .AsTable("CustomersTable")
    .Column(customer => customer.Id)
    .Column(customer => customer.Name, column => column.Header("Customer name"))
    .Build();

Excel.Write("customers.xlsx", customers, schema);
var result = Excel.Read<Customer>("customers.xlsx", schema);
```

### Validation, formulas, and templates

```csharp
var schema = Excel.Schema<Order>()
    .Column(order => order.Quantity, column => column.Range(1, 1000))
    .Column(order => order.Total, column => column
        .Formula(context => $"=A{context.Row}*B{context.Row}")
        .Format("#,##0.00"))
    .Build();

Excel.CreateTemplate("orders-template.xlsx", schema);
```

`#,##0.00` is an invariant XLSX format code: Excel with an Italian locale displays `1` as `1,00` (and `1234.5` as `1.234,50`). Use `.` for the decimal placeholder, not `,`.

`context.Row` is the physical, one-based Excel row. Do not pass a formula-looking string to `Value(...)` and expect a formula.

### Multiple worksheets and layout

```csharp
var customers = new[] { new Customer { Id = 1, Name = "Acme" } };
var customerSchema = Excel.Schema<Customer>()
    .SheetName("Customers")
    .Column(customer => customer.Id)
    .Column(customer => customer.Name)
    .Build();

Excel.Workbook()
    .AddSheet(customers, customerSchema, sheet =>
    {
        sheet.Title("Customers", "A1:D2");
        sheet.DataStartAt("A4");
    })
    .AddSheet("Notes", sheet => sheet.Note("Generated by CellSharp.", "A1:D2"))
    .Write("report.xlsx");
```

`Title`, `Section`, `Note`, and `Kpi` expand into ordinary cells, merged ranges, and styles. They do not create a separate report engine.

### Custom converters

Implement `IExcelValueConverter<TValue, TCellValue>` when a property needs an explicit scalar XLSX representation. The converter runs after scalar import conversion and before declarative/custom validation; its `Write` result is still written as a normal typed cell. See [custom converters](converters.md) for the complete contract.

## Decision guide

- Need a stable worksheet contract? Build an `ExcelSchema<T>`.
- Need a reusable import/export variation? Use `Excel.Overlay(...)`; it does not mutate the schema.
- Need import diagnostics? Inspect `ExcelReadResult<T>.Errors`; `Rows` contains valid rows by default.
- Need invalid rows retained? Pass `new ExcelReadOptions(ExcelInvalidRowPolicy.Include)` and still inspect `Errors`.
- Need native Excel validation? Use `AllowedValues`, `Range`, or `DateBetween` on a schema column.
- Need links? `Hyperlink(...)` accepts only HTTP(S) URLs or internal `#Sheet!A1` locations.

## Do not assume

- No Open XML objects belong in application code using CellSharp.
- No `.xls` reader, VBA/macro authoring, formula engine, chart API, or automatic dashboard layout exists.
- CellSharp does not fetch image URLs; download and validate bytes in application code, then call `AddImage(stream, format)`.
- The library does not sandbox caller-selected paths, custom converters, validators, formula callbacks, or row enumerables.
- The hidden `_CellSharpValidation` worksheet is an internal implementation detail; do not read it as application data.

For hostile uploads and operational limits, read the [security model](security.md). Canonical workflows in this guide are covered by `ExcelDocumentationExamplesTests`.
