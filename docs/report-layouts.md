# Report layouts

## Why report helpers exist

Use the report helpers when a worksheet needs a conventional report header, section heading, note, or KPI card. They are convenience methods over normal `ExcelRange` operations: there is no second layout engine and no special XLSX writer.

## Title

`Title(text, range)` writes the text to the top-left cell, merges a multi-cell range, and supplies a bold, larger, centered, vertically centered, wrapped starting style. It returns `ExcelRange`, so normal range styling can refine it.

```csharp
sheet.Title("Sales Performance", "B1:F2")
    .Style(style => style.FontSize(24).FontColor("#1F4E78"));
```

## Section

`Section(text, range)` is a restrained group heading: bold text, a light fill, vertical centering, and a bottom border. Like a title, it merges only when the target has multiple cells and returns `ExcelRange`.

```csharp
sheet.Section("Order detail", "A9:F9")
    .Style(style => style.FillColor("#D9EAF7"));
```

## Note

`Note(text, range)` creates visible report text in cells, with a smaller italic neutral style, wrap text, and top alignment.

```csharp
sheet.Note("Preliminary figures.", "B3:F3")
    .Style(style => style.FontColor("#666666"));
```

`Note` is visible text rendered in cells; it is not a classic Excel comment. Attach a comment to the top-left cell when both are useful:

```csharp
sheet.Note("Figures supplied by Finance.", "B3:F3");
sheet.Cell("B3").Comment("Refreshed at 08:00.");
```

## KPI cards

`Kpi(label, value, range)` requires at least two rows. Its top row is the label range and all remaining rows are the value range. Both areas are merged when they span multiple cells. The returned `ExcelKpi` exposes ordinary `Range`, `Label`, and `Value` ranges, allowing normal formula, number format, named range, style, and conditional-format operations.

```csharp
var margin = sheet.Kpi("Margin", null, "E5:F7");
margin.Value.Formula("SUM(Profit) / SUM(Revenue)")
    .NumberFormat("0.0%")
    .ConditionalFormat().LessThan(0.15)
    .Style(style => style.FillColor("#FFC7CE"));
```

Component defaults are intentionally neutral and never change row height or column width. Apply `Style(...)`, `Row(...).Height(...)`, or `Column(...).Width(...)` when the report needs a stronger visual treatment or explicit sizing. User-applied range and cell styles compose with the defaults using the normal precedence rules.

## Full report

```csharp
Excel.Workbook().AddSheet(sales, schema, sheet =>
{
    sheet.DataStartAt("A10");
    sheet.AddImage("logo.png").At("A1").Width(48);
    sheet.Title("Sales Performance", "B1:F2");
    sheet.Note("Preliminary figures.", "B3:F3");

    var revenue = sheet.Kpi("Revenue", null, "A5:B7");
    revenue.Value.Formula("SUM(RevenueData)").NumberFormat("€ #,##0");
    sheet.Kpi("Orders", 842, "C5:D7");
    sheet.Kpi("Margin", 0.173m, "E5:F7").Value.NumberFormat("0.0%");

    sheet.Section("Order detail", "A9:F9");
    sheet.Range("B11:B200").Name("RevenueData");
    sheet.Cell("B3").Comment("Source: ERP.").Hyperlink("https://example.com/sales");
    sheet.Footer.Right("Page &P of &N");
});
```

`SalesReport.xlsx`, generated by the samples project, also includes a typed table, validation, conditional formatting, and a page break.
